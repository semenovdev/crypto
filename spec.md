
# Крипто-макро торговая система V3
## Спецификация для управления портфелем SOL (и альтов)

---

## 1. Общая архитектура

Система объединяет **макро-факторы**, **ончейн-метрики** и **данные деривативов** для определения фазы цикла и оптимальной аллокации между рискованными активами и стейблкоинами.

### Источники данных

| Категория | Источник | Метрики |
|-----------|----------|---------|
| Цены и капитализация | CoinGecko | price, total_market_cap, BTC/ETH/SOL dominance |
| Стейблкоины | DeFiLlama | stablecoin_market_cap, 30d change |
| Макро (США) | FRED + Stooq | fed_rate, treasury_3m_yield, DXY, fed_balance |
| Деривативы | Binance Futures | funding_rate, open_interest, OI 7d change |
| Ончейн / valuation | Ввод пользователя | realized_price, MVRV |

---

## 2. Компоненты скоринга

### 2.1 MVRV Score (вес 25%)
Оценка фундаментальной перепроданности/перекупленности.

```
MVRV = price / realized_price
discount_to_realized = (price - realized_price) / realized_price * 100
```

| MVRV | Дисконт | Скор | Сигнал |
|------|---------|------|--------|
| < 0.8 | < -20% | 90-100 | Экстремально дешево (накопление) |
| 0.8-1.2 | -20% до +20% | 50-90 | Нейтрально-дешево |
| 1.2-2.0 | +20% до +100% | 20-50 | Нейтрально-дорого |
| > 2.0 | > +100% | 0-20 | Экстремально дорого (распределение) |

**Текущее значение:** MVRV = 0.657 → Скор = 92.84 (бычий)

### 2.2 Stable Score (вес 20%)
Оценка доступной ликвидности («сухой порошок»).

```
stablecoin_share = stablecoin_market_cap / total_market_cap * 100
```

| Stable share | 30d change | Скор | Сигнал |
|--------------|------------|------|--------|
| > 12% | > +2% | 80-100 | Сильный бычий |
| 8-12% | 0% до +2% | 50-80 | Нейтрально-бычий |
| < 8% | < 0% | 0-50 | Медвежий |

**Текущее значение:** share = 12.55%, рост = 1.14% → Скор = 81.79 (бычий)

### 2.3 Macro Score (вес 15%)
Оценка глобальной ликвидности и стоимости денег.

Факторы:
- **Fed rate + 3M yield** (базовый уровень)
- **DXY** (доллар США) — обратная корреляция с криптой
- **Fed balance change** (QT/QE)

| Условие | Эффект на macro_score |
|---------|----------------------|
| Низкие ставки / низкий 3M yield | повышают базовый macro_score |
| Высокие ставки / высокий 3M yield | понижают базовый macro_score |
| DXY > 103 | multiplicative penalty |
| DXY <= 103 | без DXY-штрафа |
| fed_balance_30d_change < -1% | QT penalty `× 0.85` |
| fed_balance_30d_change >= -1% | без QT-штрафа |

**Текущее значение:** fed_rate = 3.75%, DXY = 98.64, fed_balance_change = +0.98% → Скор = 55.0 (нейтральный)

### 2.4 Dominance Score (вес 15%)
Оценка позиции SOL относительно рынка.

```
sol_dominance — текущая доля SOL
btc_dominance — давление на альты
```

| Условие | Скор |
|---------|------|
| sol_dominance > 3% и растет | 80-100 |
| sol_dominance 1-3%, btc_dominance падает | 50-80 |
| btc_dominance > 55% и растет | 0-50 |

**Текущее значение:** SOL.D = 1.91%, BTC.D = 57.08% → Скор = 46.78 (медвежий/дефенсивный)

### 2.5 Dominance Trend Score (вес 10%)
Направление движения доминаций за 30 дней.

```
btc_dominance_30d_change — если снижается → бычий для альтов
sol_dominance_30d_change — если растет → бычий
```

**Текущее значение:** BTC.D = -0.31%, SOL.D = -0.12% → Скор = 50.61 (нейтральный)

### 2.6 TOTAL3 Score (вес 10%)
Динамика капитализации рынка без BTC и ETH.

```
total3_30d_change — рост говорит о притоке в альты
```

| 30d change | Скор |
|------------|------|
| > +10% | 80-100 |
| +5% до +10% | 60-80 |
| 0% до +5% | 40-60 |
| < 0% | 0-40 |

**Текущее значение:** +5.33% → Скор = 60.65 (нейтрально-бычий)

### 2.7 Momentum Score (вес 5%)
Краткосрочное позиционирование (деривативы).

```
funding_rate — перегрев или охлаждение
open_interest_change_7d — вход/выход капитала
```

| Условие | Эффект |
|---------|--------|
| funding_rate > 0.03% | штраф по funding |
| funding_rate > 0.10% | сильный штраф по funding |
| OI 7d change | влияет на score непрерывно через множитель `× 1.2` |

**Текущее значение:** funding = 0.0025%, OI change = -4.83% → Скор = 49.2 (слабо-медвежий)

## 3. Итоговый скор и режимы

### 3.1 Формула

```
final_score = 
    (mvrv_score × 0.25) +
    (stable_score × 0.20) +
    (macro_score × 0.15) +
    (dominance_score × 0.15) +
    (dominance_trend_score × 0.10) +
    (total3_score × 0.10) +
    (momentum_score × 0.05)
```

### 3.2 Режимы (Regime)

| final_score | Regime | Действие |
|-------------|--------|----------|
| > 80 | Risk-On / Full bullish | Максимальный риск |
| 70-80 | Mild Risk-On | Повышать риск |
| 60-70 | Accumulate / Mild Risk-On | Держать риск |
| 50-60 | Neutral / Cautious | Снижать риск |
| 40-50 | Defensive | Минимальный риск |
| < 40 | Capitulation / Risk-Off | Агрессивный дериск |

**Текущее значение:** 68.42 → `Accumulate / Mild Risk-On`

## 4. Аллокация портфеля

### 4.1 Базовое распределение

Используется ступенчатое правило:

```text
score >= 80  -> 90% risk / 10% stablecoins
score >= 65  -> 70% risk / 30% stablecoins
score >= 50  -> 50% risk / 50% stablecoins
score >= 35  -> 30% risk / 70% stablecoins
score < 35   -> 15% risk / 85% stablecoins
```

При final_score = 68.42:
- **Risk assets: 70%**
- **Stablecoins: 30%**

### 4.2 Внутри рисковой части

```
majors_pct (BTC + ETH) = risk_assets_pct × 0.6
alts_pct (SOL + прочие) = risk_assets_pct × 0.4
```

При risk = 70%:
- **Majors: 42% портфеля**
- **Alts: 28% портфеля**

### 4.3 Внутри мажоров (опционально, пока не реализовано в `crypto.py`)

```
if eth_dominance_30d_change > 0:
    BTC = 50%, ETH = 50%
else:
    BTC = 70%, ETH = 30%
```

## 5. Правила ребаланса

### 5.1 Триггеры ребаланса

| Условие | Действие |
|---------|----------|
| Текущая аллокация отличается от целевой на >10 п.п. | Ребаланс |
| Прошло 30 дней с последнего ребаланса | Плановый ребаланс, но это правило ещё не реализовано в текущем CLI |

### 5.2 Корректировка риска

| Условие | Корректировка |
|---------|---------------|
| final_score > 70 AND dominance_trend_score > 50 | Можно повысить риск |
| final_score < 60 OR funding_rate > 0.10% | Снизить риск |
| funding_rate > 0.10% | Снизить риск до 70% от базового |
| funding_rate > 0.15% | Снизить риск до 60% от базового |
| final_score < 40 | Агрессивный дериск (risk → 10-20%) |

### 5.3 Пример применения штрафа за funding

```
Базовый risk = 70%
funding_rate = 0.25% (> 0.15%)
→ risk = 70% × 0.6 = 42%
→ stablecoins = 58%
→ majors = 42% × 0.6 = 25.2% портфеля
→ alts = 42% × 0.4 = 16.8% портфеля
```

## 7. Интерпретация и действия

**Что это значит:**
- SOL фундаментально дешева (MVRV = 0.657, дисконт -34%)
- Ликвидность в стейблах есть и растет
- Макро-условия нейтральные (ставка 3.75%, DXY 98.6)
- BTC доминанта все еще высока, но снижается
- Краткосрочное позиционирование не перегрето

**Действия:**
- Держать базовую аллокацию 70/30
- Не увеличивать риск (ждем final_score > 70)
- Не снижать риск (final_score > 60, funding < 0.10%)
- Готовиться к увеличению риска при улучшении доминанты

## 10. Заключение

**Система V3 задаёт формализованный каркас управления риском и аллокацией**, но её ещё стоит считать эвристической моделью, а не полностью завершённой фондовой инфраструктурой.

Она позволяет:
- Исключить эмоциональные решения
- Обосновать каждую ребалансировку
- Адаптироваться к разным фазам цикла
- Защитить капитал в медвежьем рынке

**Уровень:** продвинутая discretionary / systematic allocation model, требующая бэктеста и операционной обвязки перед использованием на значительном капитале.
